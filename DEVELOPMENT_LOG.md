# Transposo — Development Log & Rules

This document tracks project decisions and phase-by-phase progress. It is updated at the end of every phase (or hotfix).

---

## Development Rules

1. **No commits by the AI agent.** After each phase or hotfix, a detailed commit title and body is written, but the user performs all commits.
2. **This log is updated every phase** with what was built, decisions made, and test results.
3. **Reference implementation:** the original Python project at https://github.com/arcatunc/Transposer.git (`app.py` is the full pipeline reference).
4. **Follow the roadmap** ([PROJECT_ROADMAP.md](PROJECT_ROADMAP.md)) except where the decisions below override it.
5. **Do not use '—' character in anywhere** containing codes, commit messages and comment lines.

## Agreed Decisions (override the roadmap where they conflict)

| Topic | Decision |
|---|---|
| Gemini integration | Direct REST calls with the `http` package — **not** the deprecated `google_generative_ai` package. Model: `gemini-flash-latest`, kept in a single constants file so it's easy to swap. API key via `--dart-define=GEMINI_API_KEY=...`. |
| Octave wrap | **Fixed** (differs from the Python original): transpositions crossing the B→C boundary carry the octave marker, e.g. A +4 semitones → `C#+`. |
| Instruments | Common band set grouped by key: C (Piano/Flute/Violin/Guitar) = 0, B♭ (Trumpet/Clarinet/Soprano & Tenor Sax) = +2, E♭ (Alto & Baritone Sax) = +9, F (French Horn) = +7. |
| Platform | Android first (iOS scaffolded but untested — needs a Mac). User installs Android Studio / SDK himself. |
| Localization | English + Turkish via ARB files (`flutter gen-l10n`), following system locale. |
| Note format | `Note[:Beat]` tokens, space-separated. Octave up `A+:1`, down `A-:1` (markers stackable, e.g. `C#++`). Beat defaults to 1; enharmonic input accepted (`Bb`, `Db`, `B#`…). Malformed tokens are skipped silently with a UI notice. |

**Data flow (current):** `HomeScreen` reads the text input → `transposeSequence(input, source.semitoneOffset, target.semitoneOffset)` → list of `TransposedNote` → joined `.readable` strings shown in the result card, and in parallel `buildAbcDocument(result)` → `SheetMusicView` → `renderAbc()` inside the webview draws the staff.

**Where future phases plug in:**
- *Phase 3:* a Gemini REST service under `lib/services/` (model `gemini-flash-latest`, key from `--dart-define=GEMINI_API_KEY`), fed by `image_picker`, writing its output into the note input field.
- *Phase 4:* persistence (`shared_preferences` or `hive`) + history UI; live re-render on input changes.

### ✅ Phase 2 — Measure Partitioning & ABCJS Sheet Rendering (2026-07-12)

**Built:**
- `lib/core/measure_partitioner.dart`: `partitionIntoMeasures()` joins `TransposedNote.abc` tokens, inserting `|` every 4.0 cumulative beats (with float tolerance for 0.25/0.5 sums) and a line break every 4 measures; notes longer than a measure spill and the carry keeps later bars aligned to absolute time. `buildAbcDocument()` prepends the `X:1 / M:4/4 / L:1/4 / K:C` header (L:1/4 makes beat 1 = quarter note, matching the engine's `/2`, `/4` durations).
- `assets/abcjs-basic-min.js`: ABCJS v6.4.4 bundled locally (offline-capable, no CDN).
- `assets/abcjs_template.html`: pinch-to-zoom viewport (`user-scalable=yes`, max 5x), `#paper` container, `renderAbc(abc)` JS function using `responsive: 'resize'`.
- `lib/widgets/sheet_music_view.dart`: creates a `WebViewController` (JS unrestricted), loads the template with `loadFlutterAsset`, and pushes the ABC string via `runJavaScript('renderAbc(...)')` after page load and on every widget update (`jsonEncode` escapes the string safely). When `WebViewPlatform.instance` is null (widget tests, desktop) it renders the raw ABC text instead, so the app never crashes off-device.
- `home_screen.dart`: sheet music card added below the result card; new l10n key `sheetLabel` (EN "Sheet music" / TR "Nota görünümü").
- Tests: `test/measure_partitioner_test.dart` (both roadmap scenarios, fractional-beat bar closing, 4-measure line break, spill-over, empty input, header) plus the home screen test now scrolls to and verifies the sheet card.

**Decisions:** ABCJS is bundled as a local asset rather than fetched from a CDN so sheet rendering works offline. On the spill-over case, bar lines follow absolute time: a 6-beat note produces `C6 | D2 | E4` (the bar prints after the note that crosses it; no note splitting/ties in this phase).

**Verification:** `flutter analyze` clean; 21/21 tests passing. Webview rendering itself still needs an on-device check once the Android SDK is installed (roadmap test scenario 3 of Phase 2).

#### Hotfix 2.1 — blank sheet on Android (2026-07-12)

**Symptom:** on a real Android device the transposed text appeared, but the sheet card showed only a white rectangle.

**Cause:** the template was loaded with `loadFlutterAsset` and pulled ABCJS via `<script src="abcjs-basic-min.js">`; that relative subresource fetch is unreliable inside the Android webview, so `ABCJS` was never defined and `renderAbc` silently failed.

**Fix (`sheet_music_view.dart` + `abcjs_template.html`):**
- Both assets are now read in Dart with `rootBundle.loadString`; the ABCJS source is spliced inline in place of the `<script src>` tag and the resulting single self-contained page is loaded with `loadHtmlString` (no relative URLs at all).
- The template queues the tune if `renderAbc` is called before ABCJS finishes evaluating, and flushes it on `window.load`.
- New `SheetBridge` JavaScript channel: JS errors (including "ABCJS library failed to load") are posted back to Dart, logged with `debugPrint`, and flip the widget to the raw-ABC-text fallback instead of leaving a blank square.

**Verification:** `flutter analyze` clean; 21/21 tests passing. Needs re-check on the Android device.

#### Hotfix 2.2 — staff invisible until (some) rotations (2026-07-12)

**Symptom:** after hotfix 2.1 the staff only appeared intermittently when rotating the device.

**Cause:** a layout race. The page can finish loading (and `renderAbc` can run) before Android has laid out the webview platform view, so ABCJS draws into a zero-width body: an invisible SVG. Rotation fires a `resize` that sometimes triggered a redraw, hence the intermittent behaviour.

**Fix (`abcjs_template.html` only):**
- The tune is stored in `lastAbc` and all drawing goes through `draw()`, which refuses to render while `document.body.clientWidth == 0` and instead polls every 50 ms (up to 5 s, then reports through `SheetBridge`) until the viewport has a real width.
- `window.resize` (fired on rotation) now re-renders `lastAbc` explicitly instead of relying on ABCJS's internal responsive listener, so rotations always redraw.

**Verification:** `flutter analyze` clean; 21/21 tests passing. Needs re-check on the Android device (transpose, rotate both ways, background/foreground).
